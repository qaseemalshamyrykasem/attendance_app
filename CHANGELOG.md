#### ✨ الميزات الجديدة (Features)

##### 🔐 المصادقة والأمان
- ✨ نظام تسجيل دخول بكلمة مرور
- ✨ دعم المصادقة بالبصمة (Biometric Auth)
- ✨ تشفير كلمة المرور بـ SHA-256
- ✨ جلسة مصدق عليها (Authenticated Session)
- ✨ قفل تلقائي للتطبيق

##### 👥 إدارة الطلاب
- ✨ عرض قائمة الطلاب مع البحث والفلترة
- ✨ إضافة طالب جديد
- ✨ تعديل بيانات الطالب
- ✨ حذف طالب واحد أو متعدد
- ✨ استيراد الطلاب من ملف Excel
- ✨ تصدير قائمة الطلاب
- ✨ التقاط صورة للطالب
- ✨ بحث سريع بالاسم أو الرقم الجامعي

##### 📚 إدارة المؤسسة التعليمية
- ✨ إدارة الأقسام الدراسية (Departments)
- ✨ إدارة المستويات الدراسية (Levels) - 4 مستويات افتراضية
- ✨ إدارة الشعب والفصول (Sections)
- ✨ إدارة المقررات الدراسية (Courses)
- ✨ ربط الشعب بالأقسام والمستويات
- ✨ ربط المقررات بالشعب

##### 📡 إدارة جلسات الحضور
- ✨ إنشاء جلسة حضور جديدة
- ✨ اختيار المقرر والشعبة والتاريخ
- ✨ تفعيل الجلسة وبدء خادم HTTP
- ✨ توليد رمز QR ديناميكي لكل جلسة
- ✨ عرض رمز QR لمسحه من الطلاب
- ✨ مراقبة الحضور الفعلي مباشرة
- ✨ عرض عدد المتصلين حالياً
- ✨ إيقاف مؤقت للجلسة (Pause)
- ✨ إغلاق الجلسة وإيقاف الخادم
- ✨ عرض تفاصيل الجلسة الكاملة

##### 🌐 خادم HTTP محلي
- ✨ خادم HTTP مدمج يعمل على Wi-Fi المحلي
- ✨ Endpoint لتسجيل الحضور: `POST /api/attendance/check-in`
- ✨ Endpoint لمعلومات الجلسة: `GET /api/session/info`
- ✨ Endpoint لحالة الجلسة: `GET /api/session/status`
- ✨ Endpoint للتحقق من طالب: `POST /api/student/verify`
- ✨ Endpoint لفحص الصحة: `GET /api/health`
- ✨ دعم CORS للطلبات المتقاطعة
- ✨ مصادقة بـ Session Token
- ✨ معالجة حتى 100 اتصال متزامن

#### 🔧 التغييرات (Changes)

- 🔧 استخدام **Riverpod** بدلاً من Provider لإدارة الحالة
- 🔧 استخدام **Go Router** للتنقل بدلاً من Navigator
- 🔧 استخدام **Drift** بدلاً من sqflite لقاعدة البيانات
- 🔧 استخدام **Shelf** لخادم HTTP بدلاً من dio_server
- 🔧 تحسين هيكل المشروع ليتبع **Clean Architecture**
- 🔧 فصل **Domain Layer** عن **Data Layer**
- 🔧 إضافة **Use Cases** لمنطق العمل
- 🔧 استخدام **Freezed** للـ Models غير القابلة للتغيير
